AI-Agile multi-agent plugin explained simply

Plain language overview

How the AI-Agile agent works

You have a team of AI helpers. Each one has one job. They pass a short note to each other instead of retelling the whole story every time. And before anything moves forward, one helper's only job is to ask: what if we're wrong?

The four phases

Phase 1
Figure out what to build
Before touching anything, write down what the thing is, who it's for, and what winning looks like.
Like drawing your dream treehouse before picking up a single plank.
Phase 2
Work out how to build it
Break the big idea into smaller jobs. Figure out which jobs need to happen before others. Write down the rules the builders must follow.
Like sorting out the instruction booklet before opening the LEGO box.
Phase 3
Build it, piece by piece
Pick the most important jobs. Build them. Check they work. Then pick the next batch. Testing happens alongside building — never at the end.
Like building one room of the treehouse at a time, checking it's solid before adding the next.
Phase 4
Learn and get smarter
After each build, look back at what went wrong or got confusing. Find the pattern. Update the rules so the same mistake doesn't happen twice.
Like writing notes on your treehouse drawings so next time goes better.

The rules every helper follows

Think first
Say what you're assuming. If confused, stop and ask. Never just guess and carry on.
Keep it simple
Only do what was asked. Don't add extra stuff nobody asked for. If it can be shorter, make it shorter.
Only touch your bit
Don't "fix" things you weren't asked to fix. If you see a problem outside your job, say so — don't touch it.
Know what done looks like
Before starting, write down exactly how you'll know you're finished. Vague goals mean endless loops.

The most important helper

The challenger — every phase has one
Most helpers build things. The challenger's only job is to poke holes before anything moves forward. They don't fix. They don't improve. They ask one question:
"What would have to be true for everything we just built to be completely wrong?"
If the answer is scary, everything stops until a human resolves it. If it's a small risk, it gets written down and watched. Nothing sneaks through unchallenged.

How helpers talk to each other

The short note (Handoff Schema)
Instead of each helper reading the entire conversation from the beginning, the team leader hands them a short structured note with only what they need for their job. This keeps things fast and stops helpers getting confused by information that isn't relevant to them.

Each phase runs this loop.
The challenger ★ runs in every phase — including the one that patches the rules.
